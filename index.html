Hola, curso de GIT

Iniciamos el proyecto con git init

Podemos ver el estado de los archivos con git status -s. Las interrogaciones ?? indican que los archivos no están añadidos a git y por tanto no tienen seguimiento

Para añadir los archivos utilizamos git add [nombre del archivo] Con esto añadimos el archivo al stage.


Sabremos los archivos que están en el Stage al hacer un git status -s.
Si tienen una A delante en verde es que no se han añadido al repositorio todavía.
Si tienen una M, significa que han sido modificados pero no se han añadido al Stage las modificaciones.
Si no aparece nada significa que han sido añadidos al repositorio en su última versión.

Para añadir lo que está en el stage al repositorio, usamos git commit -m "Comentario de la operación"

Si hacemos un cambio en un archivos, y queremos guardarlo en el respositorio, primero debemos añadirlo de nuevo con el git add [nombre del archivo]

Para ver los commits que hemos hecho hasta el momento usagmos git log --online Aparecerá el identificador del commit, la rama y el comentario del commit.

Si queremos deshacer todo el código hasta un commit en concreto usaremos git reset --hard [identificador del commit]

Hay que tener en cuenta que al hacer un reset --hard, todos commits posteriores a dicho commit se elimnan y por tanto todos los cambios.

Ejemplo. Si tengo los commits:

  • 1abc Cambios 1
  • 2abc Cambios 2
  • 3abc Cambios 3
  • 4abc Cambios 4
  • 5abc Cambios 5
Si hacemos el git reset --hard 3abc, es decir, al Cambios 3, vamos a perder los cambios de 5, y 4 incluidos y la aplicación se quedará con el estado correspondiente a cuando se creó el commit "Cambios 2".

Restaurar a la primera isntantanea: git reset --hard edb8b5b
